FAQ

What is Staticd?

Staticd is an ensemble of utility to deploy, manage and serve static content over HTTP.

  • staticd is a command line server managing the Staticd API service and the Staticd HTTP service.
  • staticdctl is a command line client for the Staticd API service you can use to deploy versionned content ready to be served by the Staticd HTTP service.

Where can I deploy?

You can deploy the Staticd API and HTTP services as a traditional ruby application on any PaaS supporting the ruby language or as a ruby gem utility.
Currently, the application have only been tested on Heroku, other deployment test and guides will follow.

Is this production ready?

Not yet. It's a side project and the only website in production using Staticd I'm aware of is this one. Expect bugs and alpha/beta quality releases until version 1.0.0.

How do I interact with my Staticd API service?

You can install and use the staticdctl gem which is the official Staticd client or develop your own. API documentation is available as comments in this file.
Note: The API authentication is based on the HMAC protocol.

Where are my sites stored?

Your site metadata are stored in a database. Currently the only supported database is a PostgreSQL database.
Your site resources are stored in a datastore. Currently the only supported datastore is an Amazon S3 bucket.

How do I scale?

The Staticd API and HTTP service follow the Process Model. Depending on how you choose to deploy the application, you can scale your processes using the workflow available at your PaaS or, if you've deployed Staticd on your own VPS/Private Server, just adding processes and configuring a load balancer beetween them. As the sites metadata and resources are stored in the database and the datastore, these processes doesn't need to be started on the same host.
